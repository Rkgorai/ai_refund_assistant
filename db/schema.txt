### Database Schema for AI Refund Assistant

1. `customers`
- `customer_id` (Primary Key)
- `name`
- `email`
- `phone`

2. `items` (Product Catalog)
- `item_id` (Primary Key)
- `name`
- `description`
- `category` (e.g., Groceries, Clothing, Electronics, Home Appliances)
- `base_price`
- `return_policy` (String: e.g., 'Non-Returnable', '7 Days Return', '15 Days Return', '30 Days Return', 'Replacement Only')

3. `orders` (Combined transaction & line-item table)
- `order_id` (Primary Key)
- `customer_id` (Foreign Key -> customers)
- `item_id` (Foreign Key -> items)
- `quantity`
- `order_date`
- `delivery_date` (Important: The LangGraph agent will use this at runtime to calculate if the return window has expired)
- `total_amount`
- `payment_method` (Credit Card, UPI, Store Credit)
- `refund_status` ('None', 'Requested', 'Processed', 'Rejected' - This tracks actual actions taken, rather than calculated eligibility)

4. `support_tickets` (Customer Service Logs)
- `ticket_id` (Primary Key)
- `customer_id` (Foreign Key -> customers)
- `order_id` (Foreign Key -> orders)
- `issue_description`
- `status` ('Open', 'Closed')
- `created_at`
